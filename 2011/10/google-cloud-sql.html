試用 Google Cloud SQL

Oct 12th, 2011 | Comments


Google 剛剛出的新服務 - Google Cloud SQL，可能有太多人抱怨認為 Google App Engine 的 Datastore 太難用，而且現有的 App 移植到 Google App Engine 最大問題也是 Datastore。傳統的應用程式大多數使用 Relational Database Management System (RDBMS) 關聯式資料庫管理系統作為儲存資料的系統，但 RDBMS 是比較難去擴展整個系統，起初可以靠升級系統硬件去加強效能，但硬件升級愈高，性價比愈低。而 Google App Engine 的 Datastore 是 NoSQL Database，擴展系統是一件很容易的事，可以買 5 台中級機器去組成整個系統，不夠可以再加上去，可以無限地伸延 (理論上)。NoSQL Database 是沒有 Join Table 之類的語法，你只能靠應用程式層面實現 Join Table，這會提高應用程式開發難度。

Google Cloud SQL 是用來解決傳統的應用程式移植或對使用 RDBMS 有開發經驗的程式員有多一個選擇。而 Google Cloud SQL 底層是使用 MySQL 實作的。
